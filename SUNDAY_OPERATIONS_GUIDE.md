# Sunday Operations Guide

## Before doors open

- Command Center: confirm **today’s services** count and zero critical volunteer gaps.
- Attendance: open a **SERVICE** session for the campus.
- Sunday Mode: select the correct service from the dropdown (today’s date is auto-selected).

## During service

| Task | Where |
|------|--------|
| Run sheet timing | Sunday Mode → Run sheet → Start timer / Complete segment |
| Volunteer presence | Sunday Mode → Team |
| Check-in | Attendance (or quick link from Sunday Mode) |
| Ops issues | Sunday Mode → Alerts → Log issue (inline form, not browser prompt) |
| Emergency | Sunday Mode → Emergency alert (broadcasts to ops) |

## Realtime

- Green realtime bar = connected; counts show other operators in Sunday Mode.
- If disconnected, data still saves via API; refresh or wait for auto-reconnect.

## Mobile backstage

- Use fullscreen (expand icon) for high-contrast backstage view.
- All primary actions use 44px+ touch targets.
- Prefer Team tab for quick presence updates over editing assignments mid-service.

## After service

- Complete final run sheet segment.
- Close attendance session.
- Review notifications and any logged issues in Alerts tab.

## Troubleshooting

| Symptom | Action |
|---------|--------|
| Empty run sheet | Worship Services → edit run sheet for this service |
| No volunteers on Team tab | Volunteers module → assign to event |
| Loading stuck | Retry button on Sunday Mode loading state |
| Workflow failures | Admin Center → Incidents → Replay failed workflows |
